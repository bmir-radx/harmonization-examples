# Harmonization Examples

A layered set of examples for the
[harmonization framework](../../harmonization-framework), built to be **read as
documentation** and **run as self-checking tests**.

Each example teaches a cluster of the framework's primitives on a small,
purpose-built dataset, and — crucially — explains *why* each harmonization
choice was made, not just the mechanics.

## How the examples are structured

Every built example expresses **the same harmonization three ways**, so you can
enter from whichever interface you use and see that they agree:

| File | Interface | What it shows |
|------|-----------|---------------|
| `build_rules.py` | Python API | Constructs `HarmonizationRule`s **in code**, with the rationale in comments + rule `metadata`, then `RuleSet.save()`s `rules.json`. **This is the source of truth.** |
| `rules.json` | — | Generated by `build_rules.py`. Consumed by the CLI and RPC variants. Commit it so the other two interfaces work without running Python first. |
| `run_python.py` | Python API | Loads the rules, harmonizes via `harmonize_file`, and asserts the result matches `expected_output.csv`. |
| `run_cli.sh` | JSON + CLI | Runs the `harmonize` CLI against `rules.json` — no Python needed. |
| `client.ts` | Sidecar / RPC | Drives the FastAPI sidecar through the TypeScript RPC client (in `_shared/`). |
| `input.csv` / `expected_output.csv` | — | Raw input and the pinned known-good output. |
| `README.md` | — | The harmonization story for that example. |

> **Note on interface parity.** The Python `harmonize_file` (and the sidecar)
> keep *all* input columns plus the new target columns. The CLI emits *only* the
> target columns (plus `source dataset`/`original_id` when `--include-metadata`
> is passed). This is a real, intentional difference in the framework — each
> example's README calls out which golden file each interface reproduces.

## Running the examples

These examples run against the framework installed in the sibling repo's
virtualenv. From this `examples/` directory:

```bash
# Python interface (also runs the golden-output assertion):
../../harmonization-framework/venv/bin/python 01-hello-harmonization/run_python.py

# CLI interface:
bash 01-hello-harmonization/run_cli.sh

# RPC interface (start the sidecar first, then):
node 01-hello-harmonization/client.ts
```

Each `run_python.py` prints `OK: ...` when its output matches the golden file.
